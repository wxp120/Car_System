# 车载信息娱乐系统 —— 优化创新点分析

## 一、系统性能优化

### 1.1 环境监测模块优化

**问题**: 当前 `environment.cpp` 使用 `#define DISABLE_CHART` 禁用了 Qt Charts 折线图，原因是 ARM 板上的堆内存损坏问题。

**优化方案**:
- **环形缓冲区**: 使用 QVector 作为固定大小的环形缓冲区，避免图表逐帧拷贝全量数据（`pointsVector()` 返回拷贝，对大点数开销极大）
- **数据平滑滤波**: 采用指数加权移动平均（EWMA）对传感器原始数据去噪，提高数据可靠性
- **传感器融合**: 结合温度与湿度计算体感温度（Heat Index），更直观反映人体感受
- **异常检测**: 对传感器读数做合理性校验，丢弃明显异常值（如负温度、超范围值）

### 1.2 视频采集优化

**问题**: YUYV→RGB 转换每帧执行浮点数乘法，在 ARM 上效率低。

**优化方案**:
- **查找表加速**: 预计算 YUV→RGB 的整数查找表，将浮点运算转换为查表+移位
- **零拷贝共享**: 使用共享内存传递图像数据，避免 `QImage::copy()` 的深拷贝开销
- **自适应帧率**: 根据光照条件自动调整采集帧率，低光下降至15fps节省CPU

### 1.3 网络请求优化

**问题**: 天气和地图模块无请求失败重试机制，超时无限制等待。

**优化方案**:
- **指数退避重试**: 最大3次重试，间隔 1s → 2s → 4s
- **请求超时控制**: 设置 15 秒超时，避免无限等待
- **本地缓存策略**: 天气数据缓存5分钟，地图瓦片缓存到本地文件，减少重复请求
- **并发请求管理**: 限制同时进行的网络请求数，防止大量请求耗尽连接池

### 1.4 内存管理优化

**问题**: 大量裸指针、手动 new/delete，存在内存泄漏风险。

**优化方案**:
- 使用 `std::unique_ptr` / `std::shared_ptr` 替代裸指针
- QImage 等大对象使用隐式共享特性，避免不必要的深拷贝
- 页面切换时释放非活跃页面的渲染资源

## 二、创新功能点

### 2.1 ★ CAN 总线车辆数据集成

**创新描述**: 打通车辆 CAN 总线，实时采集发动机转速、车速、冷却液温度、燃油量、故障码等车辆内部数据，替代当前模拟数据，实现真正的车联网。

**技术方案**:
- 通过 SPI 或 SocketCAN 接口与车辆 CAN 控制器通信
- 解析 OBD-II PID 协议获取标准诊断数据
- 将车速表从模拟数值改为真实 CAN 车速数据
- 增加故障指示灯（MIL）显示，实时读取 DTC 故障码
- 数据存储到 SQLite 数据库，支持历史行程回放

**关键技术栈**: SocketCAN、OBD-II PID、ISO 15765-4

### 2.2 ★ 驾驶员疲劳检测（视觉方案）

**创新描述**: 利用已有 USB 摄像头，实时检测驾驶员眼部特征，基于 PERCLOS 算法评估疲劳状态，及时预警。

**技术方案**:
- 基于 OpenCV 或 dlib 的人脸68点关键点检测
- 提取眼部区域，计算 EAR（Eye Aspect Ratio，眼睛纵横比）
- 计算 PERCLOS（Percentage of Eyelid Closure，闭眼时间占比）
- EAR 低于阈值持续 >2s 判定为疲劳，触发语音+蜂鸣器告警
- 适配夜间红外补光模式

**关键技术栈**: OpenCV、dlib、PERCLOS、EAR 算法

### 2.3 ★ 行车记录仪（Dashcam）

**创新描述**: 在倒车影像模块基础上扩展行车记录功能，支持循环录制、碰撞检测自动锁存、GPS轨迹叠加。

**技术方案**:
- H.264 硬件编码录制视频流
- 1分钟分段存储，循环覆盖旧文件
- ADXL345 加速度计检测碰撞/急刹车，自动保护关键片段
- 视频上叠加 GPS 经纬度、车速、时间水印
- 最大支持 64GB TF 卡存储

**关键技术栈**: V4L2 采集、FFmpeg / GStreamer 编码、加速度传感器

### 2.4 ★ 语音控制模块

**创新描述**: 接入离线语音识别引擎，实现多场景语音指令控制，解放驾驶员双手。

**支持指令**:
- 音乐控制：「播放」「暂停」「下一首」「上一首」「大声点」
- 导航控制：「导航到XXX」「放大地图」「缩小地图」
- 系统控制：「打开摄像头」「查看天气」「打开设置」
- 空调控制：「打开空调」「温度调到26度」「风速最大」

**技术方案**:
- 基于科大讯飞或百度语音的离线SDK
- 关键词唤醒（Wake-on-Voice），降低持续监听的功耗
- 与 QStateMachine 结合实现对话状态管理
- 反馈界面：语音识别文本实时显示在状态栏

**关键技术栈**: 语音识别SDK、关键词唤醒、状态机

### 2.5 ★ 蓝牙电话与音频

**创新描述**: 集成蓝牙 HFP（免提通话）和 A2DP（高质量音频流），实现车载免提通话和手机音乐无线播放。

**技术方案**:
- BlueZ 协议栈 + PulseAudio 音频路由
- 来电显示联系人姓名、接听/挂断按钮
- 通话记录本地保存
- 手机通讯录同步（PBAP协议）
- 蓝牙音乐元数据同步（AVRCP 1.6）

**关键技术栈**: BlueZ、HFP、A2DP、AVRCP、PBAP、D-Bus

### 2.6 OTA 远程升级

**创新描述**: 实现固件和应用层的远程无线升级（FOTA），支持增量更新和断点续传。

**技术方案**:
- Qt 应用层热更新：替换可执行文件+资源
- Linux 系统固件升级：基于 SWUpdate 或 Mender
- 增量更新：bsdiff 算法生成差分包
- 断点续传 + MD5 完整性校验
- A/B 分区回滚保护，升级失败自动回退
- 升级进度实时显示

**关键技术栈**: SWUpdate、bsdiff、HTTP Range 请求

### 2.7 多语言国际化

**创新描述**: 实现中英文界面切换，适配不同市场。

**技术方案**:
- Qt Linguist 翻译体系
- 运行时动态语言切换，无需重启
- 扩展预留日语、韩语资源

### 2.8 云端数据同步

**创新描述**: 用户偏好设置、收藏歌曲列表、常用导航地址的云端备份与多设备同步。

**技术方案**:
- Qt + MySQL/PostgreSQL 后端
- JSON Web Token (JWT) 用户认证
- 增量同步策略
- 本地 SQLite + 云端数据库双向同步

## 三、代码架构优化

### 3.1 线程安全增强

- `CaptureThread` 和 UI 线程间的 `m_currentImage` 使用 QMutex 保护（已有），增加 RAII 封装
- 所有 QAtomicInt 标志位统一使用 `std::atomic<bool>` 提升可读性
- 传感器定时器回调中避免直接操作 UI 元素，改为信号-槽方式

### 3.2 资源管理 RAII 化

- 使用 `QScopeGuard` 或标准 RAII 封装设备文件描述符的 open/close
- 网络回复的 `deleteLater()` 统一管理
- 防止定时器回调中的悬空指针引用

### 3.3 错误处理完善

- 传感器文件读取失败时累计错误计数，超过阈值提示"传感器故障"
- 网络请求失败时提供"离线模式"，使用缓存的最后有效数据
- GPS 信号丢失时的平滑处理与用户提示

### 3.4 代码复用

- 提取音乐播放器和视频播放器的公共基类 `MediaPlayerBase`
- 提取传感器模块公共接口 `ISensor`
- 统一网络请求管理类 `HttpClient`，封装重试/超时/缓存逻辑
