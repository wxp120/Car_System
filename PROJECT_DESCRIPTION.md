# 嵌入式智能车载信息娱乐系统 —— 项目描述

## 项目概述

设计并实现了一套嵌入式智能车载信息娱乐系统，基于 ARM Linux 平台，采用 Qt/C++ 构建，集成了环境感知、驾驶辅助、多媒体娱乐与在线云服务等核心功能模块。项目支持 PC 端仿真调试与 ARM 开发板真实部署的双环境编译，兼顾开发效率与嵌入式特性验证。

## 技术栈

C++ · Qt 5 Widgets · Linux 应用开发 · Linux 驱动开发 · 网络编程 · I2C/V4L2 总线

---

## 研究内容

### 1. 车载环境监测模块

基于 I2C 总线驱动 **MQ-135** 空气有害气体传感器、**DHT11** 温湿度传感器及 **AP3216C** 三合一环境光传感器（ALS 环境光 + PS 接近感应 + IR 红外），实现车内空气质量浓度、温湿度、光照强度的实时采集。通过 **Qt Charts** 曲线图进行光强度变化趋势的可视化展示，并依据有害气体浓度百分比分级变色预警（绿/橙/红），为车内环境健康提供量化依据。

### 2. 倒车影像与泊车辅助模块

通过 **V4L2** 视频采集框架驱动 USB 摄像头，采用 **YUYV→RGB 色彩空间转换** 与 MJPEG/JPEG 软解码实现多格式兼容，实时采集倒车影像。结合 **HC-SR04 超声波传感器** 实现 0.5 秒间隔实时测距，低于安全距离阈值（30cm→20cm→10cm）时触发 **蜂鸣器分级声光告警**，告警等级随距离减小逐级提升，有效提升泊车安全性。

### 3. 在线服务接入模块

- **天气服务**：调用开放天气 API 获取实时天气与 7 日预报数据，基于 JSON 解析提取温度、湿度、风向风力、PM2.5、紫外线指数等关键信息，以 **QPainter 自绘实现 5 日温度趋势折线图**，并构建省市二级联动城市选择器。
- **百度地图集成**：通过百度地图开放平台 API 实现 **GPS 实时定位**（$GNRMC 协议解析）、WGS-84 到 BD-09 坐标系纠偏转换、起终点地址 → 地理坐标正向编码、驾车路径规划、静态地图瓦片渲染与下载，支持鼠标拖拽平移与多级缩放（3-19 级）交互浏览。

### 4. 系统框架与交互设计

基于 **QStackedWidget + QPropertyAnimation** 实现多页面平滑滑动切换，通过 **QParallelAnimationGroup** 并行动画组同步当前页面滑出与目标页面滑入，配合 **OutCubic 缓动曲线** 实现自然流畅的过渡效果。包含模拟时钟、Wi-Fi 设置（SSID 扫描 + wpa_supplicant 连接 + 虚拟键盘输入）、背光亮度调节、车速仪表盘等辅助模块。

### 5. 车载多媒体娱乐模块

基于 **QMediaPlayer + QMediaPlaylist** 实现音乐播放器与视频播放器。
- **音乐播放器**：支持 MP3/WAV 媒体库自动扫描，LRC 歌词同步解析与滚动高亮，四种播放模式切换（列表循环/单曲循环/随机播放/顺序播放），收藏标记，音量飞梭控制，以及 **QPropertyAnimation 驱动的黑胶唱片旋转动画**。
- **视频播放器**：支持 MP4/MKV/WMV/AVI 多格式，播放列表管理，控制栏 10 秒无操作自动隐藏的沉浸式播放体验。

## 优化与创新点

### 创新点一：多传感器融合的环境自适应策略

将 MQ-135 空气质量、DHT11 温湿度、AP3216C 光照度三类异构传感器数据进行融合分析，通过**指数加权移动平均（EWMA）** 滤波算法抑制传感器噪声与瞬时毛刺，并结合温度与相对湿度计算**体感温度（Heat Index）**，使环境信息更贴近人体真实感受。同时引入传感器异常自诊断机制——连续三次读取失败则自动标记故障状态并提示，提升系统可靠性。

### 创新点二：YUV 色彩空间快速转换算法

针对 USB 摄像头通用的 YUYV 输出格式，设计了基于**整数乘加与预计算查找表**的色彩空间转换流水线，将传统浮点运算替换为定点整数运算，在 ARM 嵌入式平台上有显著的计算效率优势。同时实现了 MJPEG/JPEG → Qt 原生解码的零拷贝路径，避免了中间缓冲区的额外分配与拷贝开销。

### 创新点三：双线程异步采集与安全通信架构

将 V4L2 摄像头帧采集与 HC-SR04 超声波测距分别封装为独立工作线程（CaptureThread、HCSR04Thread），通过 **信号-槽跨线程通信** 与 **QMutex 互斥锁保护共享图像缓冲区**，实现采集与显示的完全解耦。线程生命周期使用 `QAtomicInt` 原子标志位控制启停，页面不可见时自动暂停采集以降低 CPU 占用，兼顾性能与功耗。

### 创新点四：多层级网络容错与离线降级策略

设计并实现了三级网络容错机制：① **指数退避自动重试**（1s→2s→4s，最多 3 次）；② **请求超时主动断开**（15 秒），避免 TCP 长连接悬挂；③ **数据本地缓存**（天气 5 分钟有效、地图文件持久化）。网络不可达时自动切换离线模式，使用缓存数据进行 UI 渲染，保证基础功能可用性，实现优雅降级。

### 创新点五：跨平台双环境编译与硬件抽象层设计

通过 Qt 工程文件的条件编译机制（`contains(QT_ARCH, arm)`），实现了 **`IS_BOARD_ENV` / `IS_PC_ENV`** 双编译宏的全代码覆盖。传感器 sysfs 路径、设备节点、视频采集、Wi-Fi 配置、背光控制、RTC 时钟同步等底层接口均通过 `#ifdef` 进行平台隔离，PC 端使用模拟数据回退。该硬件抽象层确保一套代码同时满足桌面快速迭代开发与 ARM 板端真实部署的需求。

### 创新点六：多页面手势滑动与动画引擎复用

针对车载触控交互场景，实现了**手势驱动的页面滑动导航**：通过重写 `mousePressEvent/mouseReleaseEvent` 捕获拖拽向量，阈值判向触发 `slideToPage()` 完成页面切换。动画对象（QPropertyAnimation × 2 + QParallelAnimationGroup）在首次使用时创建并在后续切换中复用，避免频繁的堆内存分配，优化了嵌入式场景下的响应延迟与内存碎片。
